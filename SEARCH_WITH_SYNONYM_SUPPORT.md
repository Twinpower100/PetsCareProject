Промпт: Реализация поиска с поддержкой синонимов через PostgreSQL ArrayField

Роль: Senior Django Backend Developer
Подсистема: PetsCare-backend

Задача:
Реализовать механизм поиска услуг в глобальном справочнике с учетом синонимов. Запрос пользователя (например, "прививка", или "укол от бешенства") должен маршрутизироваться на каноническое название (например, "Вакцинация"). Запрещено использовать внешние поисковые движки. Решение инкапсулируется на уровне PostgreSQL и Django ORM.

Требования к слою данных (Models):

Расширить модель глобального справочника услуг (GlobalService или эквивалент).

Внедрить поле search_keywords с использованием django.contrib.postgres.fields.ArrayField. Тип внутренних элементов: CharField(max_length=100). Поле не является обязательным (blank=True, default=list).

Обеспечить аппаратное ускорение поиска. Добавить GinIndex из django.contrib.postgres.indexes для поля search_keywords.

Требования к бизнес-логике (Views/Filters):

Модифицировать класс фильтрации (например, django-filter или метод get_queryset в ViewSet) для эндпоинта поиска услуг.

Внедрить предобработку поисковой строки: приведение к нижнему регистру, отсечение пробелов.

Реализовать транзакционный запрос через django.db.models.Q. Выборка формируется по условию OR:

Q(name__icontains=search_term)

Q(search_keywords__contains=[search_term]) или реализация оператора ANY для проверки вхождения подстроки в элементы массива.

Требования к интерфейсу администратора (Django Admin):

Интегрировать поле search_keywords в существующий класс кастомной админки для глобальных услуг.

Использовать исключительно нативные механизмы Django Admin для ввода ArrayField. Интеграция сторонних пакетов для тегов (taggit и аналоги) запрещена. Обеспечить ввод массива через запятую (стандартное поведение виджета ArrayField или базовое переопределение formfield_overrides), достаточное для внутреннего использования командой на этапе MVP.

Критерии приемки:

Сгенерированы и применены миграции схемы БД.

API возвращает каноническую услугу при передаче синонима в параметр search.

План выполнения SQL-запроса (EXPLAIN ANALYZE) подтверждает использование GinIndex при фильтрации по массиву, исключая Seq Scan.

Добавление, редактирование и сохранение синонимов корректно выполняется через стандартный интерфейс Django Admin без ошибок сериализации.
