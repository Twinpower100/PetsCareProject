# Отправка формы заявки

## Статус: ⏳ Не проверено

---

## Заполнение обязательных полей

- [ ] Заполнить поле "Название организации" (`provider_name`)
- [ ] Заполнить поле "Адрес организации" (`provider_address`)
- [ ] Заполнить поле "Контактный номер" (`provider_phone`)
- [ ] Выбрать минимум 1 категорию услуг
- [ ] Проверить, что все поля валидны

---

## Валидация полей

- [ ] Попытаться отправить форму с пустым `provider_name` → должна быть ошибка
- [ ] Попытаться отправить форму с пустым `provider_address` → должна быть ошибка
- [ ] Попытаться отправить форму с пустым `provider_phone` → должна быть ошибка
- [ ] Попытаться отправить форму без категорий → должна быть ошибка

---

## Отправка формы

- [ ] Заполнить все обязательные поля
- [ ] Выбрать категории услуг
- [ ] Нажать кнопку "Submit Application"
- [ ] Проверить, что отправляется POST запрос на `/api/provider-forms/`
- [ ] Проверить, что в запросе присутствуют:
  - [ ] `provider_name`
  - [ ] `provider_address`
  - [ ] `provider_phone`
  - [ ] `selected_categories` (массив ID категорий)
  - [ ] `documents` (если загружены файлы)

---

## Обработка успешной отправки

- [ ] Проверить, что после успешной отправки отображается сообщение об успехе
- [ ] Проверить, что форма очищается или происходит редирект
- [ ] Проверить, что заявка создана в базе данных со статусом `pending`

---

## Обработка ошибок отправки

- [ ] Симулировать ошибку API (неверный токен, ошибка сервера)
- [ ] Проверить, что отображается сообщение об ошибке
- [ ] Проверить текст ошибки: "Failed to submit application. Please try again."
- [ ] Проверить переводы ошибки на всех языках
